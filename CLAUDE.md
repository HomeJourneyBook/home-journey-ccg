# Home's Journey — CCG

Коллекционная карточная игра для двух игроков на одном устройстве. Tea vs Jeet, каждый защищает свою базу (20 HP).

## Структура проекта

```
CCG/
├── index.html          # Разметка: лендинг, игровое поле, экраны правил/лора/каталога, модалки
├── css/
│   └── styles.css      # Все стили
└── js/
    ├── data.js         # Данные карт — объект DEFS
    ├── abilities.js    # Система способностей: getAbilities(), triggerAbilities()
    ├── deck.js         # buildDeck(), mkCard() — сборка колоды и создание экземпляров карт
    ├── state.js        # Игровое состояние G, initState(), findC(), resetC(), lg()
    ├── render.js       # Вся отрисовка: render(), mkEl(), mkSmallEl(), reorderZones(), adjustHandOverlap()
    ├── game.js         # Игровая логика: onClick(), doAttack(), endTurn(), doBurn(), checkWin()...
    ├── catalog.js      # Каталог карт: renderCatalog(), фильтры, openCardDetail()
    └── ui.js           # UI: startGame(), showScreen(), toggleLog(), updateMulliganBtn() + boot
```

Скрипты подключаются в `index.html` в строгом порядке: `data → abilities → deck → state → render → game → catalog → ui`.

## Как добавить новую карту

Все карты описаны в `js/data.js` в объекте `DEFS`. Достаточно добавить новую запись:

```js
// Формат ключа: {faction}_{id}
// Фракция tea = золотая, jeet = фиолетовая
t_example: {
  name: "EXAMPLE",   // отображаемое имя
  cost: 2,           // стоимость в Essence
  hp: 3,             // очки здоровья (0 для заклинаний/миров/артефактов)
  atk: 1,            // атака (0 для заклинаний/миров/артефактов)
  art: "🐉",         // эмодзи-иллюстрация
  f: "tea",          // фракция: "tea" или "jeet"
  tags: ["vanguard"], // массив тегов (см. раздел ниже)
  ab: "Описание способности для игрока.",
  // Опциональные флаги типа карты (только один):
  // spell: true      — заклинание (мгновенное, идёт на кладбище)
  // world: true      — мир (перманент, один активный)
  // artifact: true   — артефакт (перманент, до 2 штук)
  // unique: true     — легендарная карта (1 экземпляр в колоде)
},
```

После добавления карту нужно включить в колоду в `js/deck.js` — в функцию `buildDeck()`, в нужный массив (`weak`, `legs`, `spells`, `worlds`, `arts`, `extra`).

Уникальные карты с особой механикой, не покрытой тегами, требуют явного switch-кейса в `getAbilities()` в `js/abilities.js`.

## Система тегов

Теги — это строки в массиве `tags` карты. Они могут быть простыми (`"vanguard"`) или с числовым значением (`"heal:2"`).

### Все доступные теги

| Тег | Значение |
|-----|----------|
| `vanguard` | Может атаковать в тот же ход, когда выходит на поле |
| `provoke` | Все атаки врага должны быть направлены на эту карту |
| `pierce` | Игнорирует Provoke, может атаковать базу напрямую |
| `fear` | При атаке: цель пропускает следующий ход |
| `burn` | При атаке: цель загорается (−1 HP в начале каждого хода) |
| `heal:N` | Активная способность: вылечить союзника на N HP и снять дебаффы |
| `aoe:N` | Активная способность: нанести N урона ВСЕМ врагам |
| `draw:N` | Добрать N карт (тайминг зависит от типа карты — см. ниже) |
| `regen:N` | В начале хода: восстановить себе N HP |
| `revive:full` | Мгновенно: воскресить последнее существо с кладбища (полное HP) |
| `revive:any` | Модификатор к `revive:full`: воскрешает с любого кладбища |
| `bounce` | Мгновенно: вернуть все карты с поля в руки владельцев |
| `maxhp_add:N` | Добавить N к максимальному HP (тайминг зависит от типа карты) |
| `spell` | Флаг типа: заклинание |
| `world` | Флаг типа: мир |
| `artifact` | Флаг типа: артефакт |
| `unique` | Флаг типа: легендарная карта |

### Как тег `draw` определяет тайминг

Тайминг тега `draw` автоматически определяется в `getAbilities()` по типу карты:

- `spell` → `instant` (сразу при розыгрыше)
- `world` или `artifact` → `on_turn` (в начале каждого хода)
- `unique` → `on_turn` (в начале каждого хода)
- обычное существо → `on_attack` (при каждой атаке)

Аналогично работает `heal`: для `artifact`/`world` — `on_turn` (лечит всех союзников), для существа — `active` (ручная активация).

## Система способностей

### getAbilities(card) → [{timing, effect, val, ...}]

Разбирает теги карты и возвращает массив объектов способностей. Каждый объект содержит:

- `timing` — когда срабатывает
- `effect` — что делает
- `val` — числовое значение (если есть)
- `target` — `'all'` для массовых эффектов
- `self` — `true` для эффектов на саму карту (regen)
- `any` — `true` для `revive` с любого кладбища

### Тайминги (timing)

| Тайминг | Когда срабатывает |
|---------|-------------------|
| `passive` | Постоянный эффект, пока карта на поле |
| `instant` | Сразу при розыгрыше (заклинания) |
| `on_enter` | При выходе на поле |
| `on_turn` | В начале хода владельца |
| `on_attack` | При каждой атаке |
| `on_kill` | При убийстве вражеской карты |
| `active` | Ручная активация игроком |

### triggerAbilities(card, timing, ctx)

Вызывается из `game.js` и `endTurn()`. Фильтрует способности по таймингу и выполняет эффекты. `ctx` — контекст: `{target}` для атак и лечения.

Вызовы в коде:
- `triggerAbilities(card, 'instant')` — при розыгрыше заклинания
- `triggerAbilities(card, 'on_turn')` — в `endTurn()` для миров и артефактов
- `triggerAbilities(card, 'on_enter')` — при выходе существа (Faeron, Maltor)
- `triggerAbilities(card, 'active')` — при нажатии кнопки AOE

### Эффекты (effect)

| Эффект | Что делает |
|--------|-----------|
| `aoe` | Урон всем врагам на N |
| `burn` | Поджигает цель (`ctx.target`) |
| `fear` | Пугает цель (`ctx.target`) |
| `draw` | Добирает N карт текущему игроку |
| `hp_add` | Лечит: цель / всех союзников / саму себя (regen) |
| `hp_all` | Увеличивает maxHP и HP всех союзников (Aslex) |
| `hp_base` | Восстанавливает HP базы (Reaper) |
| `maxhp_add` | Увеличивает максимальный HP цели или всех |
| `atk_all` | Даёт бонус ATK всем союзникам (Tuborg) |
| `bounce` | Возвращает все карты с поля в руки |
| `revive` | Воскрешает существо с кладбища |
| `salvage` | Возвращает карту с кладбища в руку |
| `raise` | Воскрешает последнюю карту с любого кладбища с 1 HP (Phlegmor) |
| `ess_max` | Увеличивает максимум Essence |
| `ess_add` | Добавляет Essence |

## Игровое состояние (G)

Глобальный объект `G` в `js/state.js`:

```js
G = {
  turn: 'tea' | 'jeet',   // чей сейчас ход
  turnNum: Number,          // номер хода
  phase: 'action' | 'selectTarget' | 'healTarget' | 'burn',
  sel: cardId | null,       // выбранная карта
  previewCard: cardId | null,
  jeetFirstTurn: Boolean,
  logs: [{msg, cls}],
  mulligan: { tea: {used}, jeet: {used} },
  tea: PlayerState,
  jeet: PlayerState,
}

PlayerState = {
  hp, maxHp,
  ess, essMax,
  hand: [Card],
  field: [Card],
  deck: [Card],
  grave: [Card],
  void: [Card],
  world: Card | null,
  artifacts: [Card],
  extraDraw: Number,
  burned: Boolean,
}
```

## Фазы игры (G.phase)

- `action` — обычный ход: можно играть карты, выбирать существ для атаки
- `selectTarget` — существо выбрано, ждём цель для атаки или удара по базе
- `healTarget` — Orbiton выбран, ждём союзника для лечения или врага для атаки
- `burn` — ждём карту из руки для сожжения (увеличение Essence)
